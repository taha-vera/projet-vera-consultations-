VERA v2 (Model A+) — Complete End-to-End Encrypted Pipeline
================================================================================

                        ┌─ VERA CLIENT ─┐
                        │   (Android)    │
                        └────────────────┘
                               │
                    1. Fetch TEE Identity
                    ├─ pk_tee
                    ├─ quote (SGX)
                    └─ mrenclave
                               │
                    2. Verify Quote Locally
                    ├─ Check DCAP signature
                    ├─ Verify mrenclave
                    └─ Timestamp fresh (<5min)
                               │
                    3. Load/Increment Nonce Counter
                    ├─ client_session_id = sha256(pk_client)
                    ├─ nonce_counter += 1 (persistent)
                    └─ Generate nonce = counter || timestamp_ms || random(16)
                               │
                    4. Sign Entire Message
                    ├─ message_to_sign = {version, payload, encapsulated_key,
                    │                      ciphertext, timestamp_client, nonce,
                    │                      client_session_id}
                    └─ signature = Ed25519.sign(message_to_sign, sk_client)
                               │
                    5. HPKE Encrypt
                    ├─ aad = SHA256("VERA_v2" || pk_tee || mrenclave)
                    ├─ plaintext = {user_hash, artist_id, timestamp, device_id}
                    └─ (ciphertext, encapsulated_key) = hpke_seal(pk_tee,
                                                                    plaintext,
                                                                    aad)
                               │
                    ═══════════════════════════════════════════════════════════
                    PLAINTEXT DESTROYED HERE (client-side)
                    ═══════════════════════════════════════════════════════════
                               │
            ┌──────────────────┴──────────────────┐
            │  ENCRYPTED OVER HTTPS (TLS 1.3)    │
            │  {ciphertext, encapsulated_key,    │
            │   nonce, signature, client_id}     │
            └──────────────────┬──────────────────┘
                               │

                    │   CDN (Akamai)      │ ← BLIND (ciphertext only)

                               │
                    ┌─────────────────────┐
                    │ Service Mesh        │ ← BLIND (no tap filters)
                    │ (Istio/Envoy)       │   No sidecar logging
                    └──────────┬──────────┘
                               │
                    ┌─────────────────────┐
                    │  VERA Collector     │ ← BLIND FORWARDER
                    │  (No decryption)    │   Forwards ciphertext to TEE
                    │  (No logging)       │   Metadata only: size, latency
                    └──────────┬──────────┘
                               │
            ┌──────────────────┴──────────────────┐
            │  ENCRYPTED INSIDE TEE VPC           │
            │  (AMD SEV-SNP or Intel SGX)         │
            └──────────────────┬──────────────────┘
                               │
                    ┌──────────────────────────┐
                    │  VERA TEE ENCLAVE        │
                    │  (Plaintext ONLY here)   │
                    └──────────────────────────┘

        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
    6. Verify                  │                  7. Aggregate
    ├─ DCAP signature OK       │                  ├─ Add to DP accumulator
    ├─ Client signature OK     │                  ├─ Update epsilon budget
    ├─ Nonce NOT seen before   │                  └─ Maintain running state

    ├─ Timestamp monotonic     │

        ┌──────────────────────┼──────────────────────┐
        │                      │                      │

    ├─ plaintext.fill(0xAA)    │                  ├─ result = {aggregate,
    ├─ plaintext.fill(0x55)    │                  │            n, timestamp,

KEY TRANSITIONS (Model A vs Model A+)
================================================================================

MODEL A (Uncorrected):
- HPKE context as separate parameter ❌
- Quote not bound to pk_tee ❌
- Signature covers only payload ❌
- Client TTL not enforced ❌
- Nonce not monotonic ❌
- Plaintext destruction incomplete ❌

MODEL A+ (Corrected):
- HPKE context moved to AAD (cryptographically bound) ✅
- Quote binds to pk_tee || timestamp || nonce_quote ✅
- Signature covers entire message (version, payload, keys, nonce, id) ✅
- Client enforces pk_tee_valid_until (refetch if expired) ✅
- Nonce = counter || timestamp_ms || random (monotonic client-side) ✅
- Plaintext destruction: triple-pass fill + zeroize + lfence ✅

CRYPTOGRAPHIC INVARIANTS (CCS-Grade)
================================================================================

┌─────────────────────────────────────────────────────────────────────────┐
│ INVARIANT 1: Plaintext Confinement                                      │
│ Plaintext exists ONLY inside TEE enclave memory.                        │
│ Proof: HPKE IND-CCA ⟹ ciphertext opaque without sk_tee                │
│        sk_tee sealed in TEE ⟹ only verified TEE decrypts               │
│        attestation verified by client ⟹ only legitimate TEE proceeds   │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ INVARIANT 2: Anti-Replay (Triple Defense)                               │
│ Attacker cannot replay valid ciphertext:                                │
│ 1. Counter monotonic per client (stored persistently)                   │
│ 2. Timestamp monotonic per client (TEE tracks)                          │
│ 3. Exact nonce uniqueness (server-side cache, 1h TTL)                  │
│ Violation requires compromising client storage + server state           │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ INVARIANT 3: Cryptographic Binding (No Substitution)                    │
│ Quote ← SHA256(pk_tee || timestamp || nonce_quote)                     │
│         (attacker cannot forge quote + alternate pk_tee)                │
│ Signature ← Ed25519(entire_message including nonce)                    │
│            (attacker cannot replay ciphertext into new request)         │
│ HPKE AAD ← SHA256(version || pk_tee || mrenclave)                      │
│           (ciphertext bound to specific TEE version)                    │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ INVARIANT 4: Attestation Verification (Client-Side)                     │
│ Client MUST verify:                                                     │
│ ✅ DCAP signature on quote (Intel/AMD responsibility)                   │
│ ✅ mrenclave matches known-good hash (from GitHub releases)             │
│ ✅ Timestamp freshness (< 5 minutes)                                    │
│ ✅ Quote binding to pk_tee (cryptographic)                              │
│ Failure ⟹ ABORT (reject entire request)                                │
└─────────────────────────────────────────────────────────────────────────┘
